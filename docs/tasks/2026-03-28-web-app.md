# Web app — host UI as a standalone web page

## Goal

Move the UI out of the extension page to a hosted web page (e.g. Cloudflare Pages). The extension becomes invisible infrastructure — its only job is injecting a content script into YouTube iframes so the web page can make same-origin YouTube API calls via postMessage RPC.

This gives the YouTube embed player for free (real HTTP origin, no Referer hacks), and makes the tool accessible without navigating to `chrome-extension://`.

## Why it works with minimal changes

The current `dist/ext/index.html` is already almost a standalone web app:

- `worker.js` — pure mediabunny, no extension APIs, already bundled alongside `index.html`
- `initContentRpc` — creates a hidden `youtube.com/embed/` iframe; content script is injected into it (because `all_frames: true` applies to any parent page, not just the extension page); postMessage back to parent uses `"*"` so works from any origin
- `downloadFormat`, `fetchThumbnail` — run inside the youtube.com iframe context, unaffected by parent origin
- File download via blob URL — works anywhere

## Gaps to close

### 1. Extension detection (code change)

`initContentRpc` currently waits forever for `ytdl-ready` if the extension isn't installed. Need a timeout + clear "install extension" prompt instead of a generic error.

### 2. CSP header for WASM (deploy config)

The `wasm-unsafe-eval` CSP is currently enforced via `manifest.json`. On a hosted page the server needs to send it. On Cloudflare Pages this is a `_headers` file:

```
/assets/*
  Content-Security-Policy: script-src 'self' 'wasm-unsafe-eval'
```

### 3. Hosting / deploy pipeline (infra)

Follow `../zipview` wrangler setup — static assets served via Cloudflare Workers + Assets:

```jsonc
// wrangler.jsonc
{
  "name": "yt-dlp-ext",
  "assets": {
    "directory": "./dist/ext",
    "not_found_handling": "single-page-application",
  },
}
```

Add `"release": "wrangler deploy"` script. No separate build target needed — `dist/ext/` output is self-contained.
